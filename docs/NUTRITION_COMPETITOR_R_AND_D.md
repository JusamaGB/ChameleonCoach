# Nutrition Core Competitor R&D

## Purpose

This document narrows the earlier broad platform R&D into one practical question:

`What nutrition and hybrid coaching features appear central to regular use in leading coaching platforms, and which of those should shape Chameleon Nutrition Core?`

The goal is not to copy feature lists. The goal is to identify:

- what coaches appear to use repeatedly
- what clients appear to interact with repeatedly
- what is likely sticky enough to justify product surface area
- what is probably sales-page noise rather than daily workflow value

---

## Research Focus

Platforms reviewed in this pass:

- ABC Trainerize
- TrueCoach
- Everfit
- Healthie
- Practice Better
- Kahunas

Priority lens:

- nutrition coaching
- PT + nutrition hybrid coaching
- coach-to-client accountability loops
- repeated weekly use, not edge-case admin tools

---

## Pattern Summary

Across these tools, the most repeated nutrition-related usage patterns are:

1. meal guidance or meal-plan delivery
2. food logging or meal journaling
3. macro or nutrition target tracking
4. habit tracking
5. check-ins and accountability forms
6. messaging / comments around logged data
7. progress review through simple charts or summaries

That is the clearest recurring cluster.

What shows up less consistently as a daily core:

- advanced recipe engines as the main product center
- deeply clinical workflows unless the product is healthcare-oriented
- heavy automation as the first thing sold to coaches

So the likely sticky Nutrition Core is not “big meal planning software.”
It is:

`delivery + logging + accountability + review`

## Competitor Findings

## ABC Trainerize

Observed repeated emphasis:

- meal plans
- macro tracking
- habit coaching
- check-in forms
- messaging
- progress review

Why it matters:

- Trainerize treats nutrition as part of ongoing accountability, not just static plan delivery
- habits and check-ins are positioned as repeated coach/client touchpoints
- it strongly reinforces hybrid coaching, where nutrition sits beside training in the same workflow

Likely high-usage features:

- habit tracking
- check-in forms
- macro adherence
- meal guidance
- in-app messaging around adherence and progress

Implication for Chameleon:

Nutrition Core should not stop at meal-plan rows.
It should include:

- nutrition habits
- nutrition check-ins
- coach review context

## TrueCoach

Observed repeated emphasis:

- meal plans via documents
- nutrition goals
- habit tracking
- MyFitnessPal integration
- workouts first, nutrition second

Why it matters:

- TrueCoach still frames nutrition as a meaningful extension, but not as deeply as Trainerize
- it suggests that for PT-first coaches, nutrition support often means lightweight coaching structure rather than a fully separate dietitian workflow

Likely high-usage features:

- nutrition targets
- meal-plan delivery
- habits
- comments/messaging around adherence

Implication for Chameleon:

For hybrid PT coaches, Nutrition Core should include a lightweight path that feels integrated with the rest of coaching, not like switching into a separate clinical product.

## Strongest Repeated Features Across Competitors

The features that appear most consistently across these platforms are:

### 1. Nutrition plan delivery

This can be meal plans, macro targets, nutrition goals, or structured protocols.

Conclusion:

Chameleon Nutrition Core needs a stronger coach-side delivery layer than just editing one client meal sheet manually.

### 2. Food logging / meal journaling

This appears constantly in nutrition-first and hybrid tools.

Conclusion:

Client nutrition logging is one of the strongest candidate features for real recurring use.

### 3. Habit tracking

This appears repeatedly in Trainerize, Kahunas, and broader hybrid coaching workflows.

Conclusion:

Nutrition Core should likely include habit tracking as a first-class accountability feature.

### 4. Check-ins

Check-ins are repeatedly presented as the bridge between plan delivery and coaching personalization.

Conclusion:

Nutrition Core should likely include nutrition-focused check-ins, not just meal rows and notes.

### 5. Messaging / comments around logged behavior

Repeatedly used as the human layer that keeps clients engaged.

Conclusion:

Even if full messaging remains out of scope platform-wide, Nutrition Core should support a visible feedback loop tied to client nutrition entries and check-ins.

### 6. Progress visibility

Charts, adherence summaries, or trends repeatedly show up as part of the review loop.

Conclusion:

Coach review and client-facing visibility both matter, but the first priority is simple readable review, not advanced analytics dashboards.

---

## Likely Low-Priority or Later Features

These may still matter later, but they do not look like the first things to build if we are optimizing for real coach usage:

- overly advanced recipe engines
- deep food database architecture as the first differentiator
- highly clinical charting workflows
- complex automation-first nutrition systems
- AI-heavy nutrition tooling before the core accountability loop exists

---

## Chameleon Nutrition Core Recommendation

Based on this pass, Nutrition Core should probably be shaped around the following stack:

### Coach-side

1. recipe or meal library
2. meal-plan templates
3. nutrition habit templates
4. nutrition check-in template/questions
5. client nutrition assignment / application flow

### Client-side

1. meal plan view
2. food or meal logging
3. nutrition habit tracking
4. weekly nutrition check-in
5. nutrition notes/reflections

### Coach review

1. nutrition adherence overview
2. recent logs and check-ins
3. habit completion visibility
4. coach notes / follow-up context

That is the clearest competitor-backed path toward a Nutrition Core that feels used, not decorative.

---

## Suggested Build Order

Recommended next sequence for Nutrition Core:

1. coach recipe / meal library
2. meal-plan templates
3. template-to-client application flow
4. nutrition habits
5. nutrition check-ins
6. coach review surface
7. client logging refinements

This keeps the build focused on the repeated usage loops competitors seem to depend on most.

---

## Key Strategic Takeaway

The best nutrition products are not just “meal plan builders.”

The repeated sticky loop is:

`coach assigns nutrition guidance -> client logs / checks in -> coach reviews and responds -> client stays accountable`

So Chameleon Nutrition Core should be built around:

- delivery
- logging
- habits
- check-ins
- review

That is the strongest signal from this competitor pass.
